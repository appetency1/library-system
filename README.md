# 图书馆座位预约系统

这是一个面向高校图书馆场景的座位预约管理系统，可用于计算机专业毕业设计、课程设计和本地演示。系统采用前后端分离架构，学生可以自助预约座位并完成签到签退，管理员可以维护阅览室、座位、用户、预约记录和系统公告。

## 系统功能清单

### 学生端

- 学生注册、登录和退出登录
- 查看系统公告和个人预约概览
- 按阅览室、日期、时间段查询座位状态
- 创建座位预约
- 查看个人预约记录
- 按状态、日期范围和关键词筛选预约
- 取消未开始的预约
- 对到达开始时间的预约进行签到
- 对已签到的预约进行签退

### 管理端

- 管理员登录和退出登录
- 查看统计看板，包括今日预约数、签到率、座位使用率和预约趋势
- 阅览室管理：新增、编辑、启用、停用阅览室
- 座位管理：新增、编辑、启用、停用座位
- 预约管理：筛选、查看并强制取消异常预约
- 用户管理：查看用户并控制账号启用状态
- 公告管理：新增、编辑、发布、下线公告

### 后端能力

- REST API 接口
- JWT 身份认证和角色权限控制
- SQLite 本地数据库
- 演示数据自动初始化
- 预约冲突校验
- 预约状态流转校验
- 后端单元测试和接口集成测试

## 技术栈

- 前端：Vue 3、Vite、TypeScript、Pinia、Vue Router、Axios、ECharts
- 后端：Node.js、Express、TypeScript、SQLite、JWT、bcryptjs
- 测试：Vitest、Supertest

## 运行步骤

### 1. 安装依赖

```bash
npm install
```

### 2. 启动前后端开发服务

```bash
npm.cmd run dev
```

如果不是 Windows 环境，也可以使用：

```bash
npm run dev
```

默认访问地址：

- 前端页面：http://localhost:5173
- 后端接口：http://localhost:3000

### 3. 单独启动服务

```bash
npm.cmd run dev:web
npm.cmd run dev:api
```

### 4. 测试和构建

```bash
npm.cmd test
npm.cmd run build
```

## 数据库说明

后端默认使用 SQLite，本地数据库文件路径为：

```text
data/library-seat-reservation.sqlite
```

首次启动后端时，系统会自动创建表结构并写入演示数据，包括默认账号、阅览室、座位和公告。删除该数据库文件后重新启动后端，会重新初始化演示数据。

主要数据表：

- `users`：用户账号、角色和状态
- `rooms`：阅览室信息
- `seats`：座位信息
- `reservations`：预约记录和签到签退状态
- `notices`：系统公告
